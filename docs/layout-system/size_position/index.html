尺寸与位置

盒模型（尺寸）​

盒模型 空间组成​

content-box组成
  1. content 宽高
  2. padding 上右下左
  3. border 上右下左
  4. margin 上右下左
注意
  • padding 不能为负值 margin 可以为负值
  • 自身的背景会平铺到 非 margin 区域
  • margin-top 的传递： 子盒子 margin-top 传递给父盒子 margin-top
  • margin上下折叠 取最大值

块级盒子 内联盒子​

块级盒子
  • display: block
  • 独占一行
  • 支持 所有样式
  • 不写宽度 盒子横向空间与父容器width相同 内容会自适应
  • 所占区域为矩形
纯内联盒子
  • display: inline
  • 不独占一行 在空间足够时不会换行
  • 不支持 部分样式 如 width height
  • 宽度由内容决定
  • 所占区域不一定是矩形 如 换行前后
  • 内联盒子间 会有空隙
布局
  • 尽量使用块级元素
  • 尽可能不用内联元素
    • 内联元素 只用在 文本修饰

自适应盒模型​

  • diaplay:block
  • 不写width 盒子横向空间与父容器width相同 且内容会自适应
  • 默认 box-sizing: content-box

标准与怪异 盒模型​

区别
  • 标准盒模型 宽高为content区域
    • box-sizing: content-box
  • 怪异盒模型 宽度为content+padding+border
    • box-sizing: border-box
    • button / bootstrap框架 就是默认怪异盒模型
怪异盒模型 应用
  1. 量取好UI设计稿具体尺寸后 不再去calc计算
  2. 需要设置百分比宽度的盒子模型

浮动（位置）​

浮动详解

元素浮动 会脱离文档流 并按浮动方向 元素外边界贴到 其他浮动元素外边界或父元素内边界为止

文档流：元素在web页面上的呈现方式 按照出现的先后顺序进行排列 如二维图片沿着三维高度方向飘起一定高度

清除浮动
.clearfix::after{
    content : "";
    clear:both;
    display:block;
}
浮动特性
  • 只会影响后面的元素 不影响前面的元素 （同级）
  • 文本不会被浮动元素覆盖 文字会围绕浮动的元素
  • 具备内联盒子特性 宽度不设置 则由内容来决定 如 内容是一段文本，则宽度默认为文本内容宽度
  • 具备块级盒子特性 支持所有样式
  • 浮动元素排不下 自动换行

定位（位置）​

定位详解
  • position 元素在文档中的定位方式
  • position: static 默认值
  • position: relative
    • 还在文档流中 且不影响其他元素
    • 相对自己原来的位置 为参照
  • position: absolute
    • 脱离文档流 且不占空间
    • 具备内联盒子特性 宽度不设置 则由内容决定
    • 具备块级盒子特性 支持所有样式
    • 以外层最近定位元素 为参照 最近非static定位祖先元素
    • 没有定位祖先 则以可视区 为参照 不是body
  • position: fixed
    • 与absolute其他特性类似
    • 以可视区 为参照
  • top / right / bottom / left 决定具体位置
定位注意
  • position: sticky
    • 未触发特定阀值 为相对定位 relative
    • 触发特定阀值后 为固定定位 fixed
    • 特定阀值 以可视区 为参照 是否触发
  • z-index 失效
    • 优先与同级元素z-index比较 而非同级元素的子元素z-index比较

扩展​

writing-mode​

文字书写方向
  • horizontal-tb 水平方向自上而下的书写方式
  • vertical-rl 垂直方向自右而左的书写方式
  • vertical-lr 垂直方向自左而右的书写方式
  • sideways-rl 内容垂直方向从上到下排列
  • sideways-lr 内容垂直方向从下到上排列

BFC​

概念

BFC即Block Formatting Contexts(块级格式化上下文)

可以看作是隔离后的独立容器，容器里元素不会在布局上影响到外面的元素。

触发条件 满足任意一个
  • float 的值不是none 脱离文档流
    • 浮起来了
  • position 的值不是static或者relative 脱离文档流
    • absolute fixed
  • display 的值是inline-block、table-cell、flex、table-caption或者inline-flex
  • overflow 的值不是visible
BFC应用
  • 避免 margin-top 的传递
  • 避免 父容器 高度塌陷
  • flex 和 grid 默认自带BFC规范

默认样式 清除方案​

默认样式与设计图冲突

一些HTML标签会有默认样式，如：body会有margin:8px；ul会有margin:16px 0;及padding-left:40px。

切图软件中进行尺寸或位置测量完对应到样式时，可能会受到默认样式的影响，从显示效果跟设计图效果不符。

清除默认样式方案
  • Reset CSS
  • Normalize CSS
  • 混合2者
